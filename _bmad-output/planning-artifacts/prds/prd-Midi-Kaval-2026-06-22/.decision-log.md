# Decision Log — Security & Data Protection PRD

## 2026-06-22 — PRD Initialization

| # | Decision | Rationale | Author |
|---|----------|-----------|--------|
| 1 | PRD scope limited to OWASP-identified security gaps | The brainstorming session (2026-06-22-0836) produced 25 requirements across 4 priority tiers. All are included. | Admin |
| 2 | Backward compatibility as a hard constraint | User explicitly stated existing functionality must not break. Added NFR-SEC-01 and acceptance criteria accordingly. | User |
| 3 | Encryption at the EF Core value converter layer | Encrypting at the application layer (not DB-level TDE) keeps encryption transparent to queries, DTO mapping, and search — preserving backward compatibility. | Architect |
| 4 | AES-256-GCM as encryption algorithm | .NET 8 provides `AesGcm` natively. GCM provides authenticated encryption (integrity + confidentiality). Non-deterministic for contact/GPS, deterministic (AES-SIV) option for searchable name field — Open Question 1. | Architect |
| 5 | CSP from backend middleware + index.html meta tag | Two-layer defense: if the API is proxied and headers stripped, the meta tag still enforces policy. | Architect |
| 6 | Rate limiting via ASP.NET Core 8 built-in middleware | `Microsoft.AspNetCore.RateLimiting` is part of the framework — no third-party dependency needed. Role-based exemption requires the `PolicyName` + `RequireAuthorization` pattern. | Architect |
| 7 | Anonymization (not hard-delete) for retention | Project-context.md explicitly prohibits hard-deleting cases. Anonymization preserves operational data while removing PII. | Project rule |
| 8 | 7 stories identified for MVP | Each story maps to a coherent, independently deployable change. Critical (Encryption, CSP, Audit Redaction) and High (Rate Limiting, Retention, Erasure, Import Validation) are all included. | PM |
